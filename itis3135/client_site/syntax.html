Syntax Overview

CSS syntax refers to the structured way we write styles to apply to HTML elements. At its core, each rule is composed of a selector and a declaration block. The selector targets the HTML element or elements we want to style, and the declaration block contains one or more declarations enclosed in curly braces. These declarations define the actual styles, such as color, font size, or margin. Each declaration itself is made up of a CSS property, a colon, the value to be assigned to that property, and ends with a semicolon. Learning this format is the foundation of writing effective CSS. h1 { color: red; font-size: 24px; }

Declarations

In CSS, declarations define how the targeted HTML elements should appear. Each declaration follows a consistent format: it starts with the property you want to modify, followed by a colon, then the value you want to assign to that property, and ends with a semicolon. Properties can define visual features like text color, background styles, and borders, or they can manage layout behavior, such as spacing, display types, and positioning. The key to writing good CSS lies in understanding which properties to use and how different values affect the rendered result. Declarations are grouped within curly braces and can be as short as a single line or span multiple lines for clarity. p { margin: 20px; line-height: 1.6; }

CSS Selectors

CSS selectors are essential tools that let us pinpoint exactly which HTML elements to style. There are many types of selectors available. Basic selectors include type selectors that target elements like paragraphs, class selectors that match elements with a specific class, and ID selectors that apply styles to elements with a specific ID. More advanced selectors include attribute selectors, which style elements based on the presence or value of an attribute, as well as pseudo-classes and pseudo-elements which style elements based on state or specific content. The wide range of selectors gives developers powerful tools to write clean, specific, and scalable CSS. .highlight { background-color: yellow; }

CSS Comments

CSS comments allow developers to add notes and explanations within their stylesheets. These notes are not displayed in the browser and serve purely as guidance for other developers or for the original author when revisiting code. Comments are enclosed within a specific syntax: they begin with a forward slash and asterisk and end with an asterisk and forward slash. This allows for multi-line comments or short inline comments next to declarations. Comments are especially useful in large projects to label sections, explain complex logic, or temporarily disable rules without deleting them. /* This is a CSS comment */
